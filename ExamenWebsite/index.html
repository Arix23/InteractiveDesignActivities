Websites

Websites are an important element for many different companies, websites allow them to have a place where they can put their content and people can interact with it easily.


Frontend
The Front End is everything the end user sees on the website, the part that is presented to the public. It handles the design of the website and the things that interact with the user.

Backend
The Back End is the system behind the website, the thing that allows the website to work correctly behind the scenes, that handles the servers and the data.




Who is the End User?
The End User is the one who uses the website or the final product, the main target. Usually they don't have technical knowledge to know exactly how it works.
Who is the Client?
The client is the one who buys the product, the one that pays the money for the website to be developed. The client isn't the one who uses the website, but they have the idea that they want to present to the end user.

CDN Architecture
A CDN helps your website load faster, since it saves a stored version of your website in several servers that are located in different places, so the user can access the static content of the website faster, so the user doesn't have to wait for the whole page to load. The static content is delivered at a high speed to the user.


History of HTML and CSS
  • HTML started as a way to share information between websites, it was purely text based, it was thought that people would only share scientific information using the Internet.
  • After some time, HTML added image tags, but they were limited.
  • In 1997, designs started to appear on HTML, it became possible to change color, font, background, sizes, etc.
  • HTML has become the backbone of web development in the latest years.
  • CSS started as a way to separate design from content in December 1996.
  • CSS became the best method to give websites a consistent look that worked on different browsers.
